Button styles

A <button> element should be used whenever an action is performed.

<button> elements sit in the tab order of the page and can be activated with the 'space' or 'enter' keys. With elements that appear in the tab order the user's browser will provide an 'outline' effect to highlight its position on the page, this is an accessibility feature and should not be disabled.

Angular Material buttons

The Angular theme file nbs-material.theme in combination with the mat-button.overrides file sets up the styles needed for buttons in your app.

Implementation

You declare a button by using the <button> tag and adding the 'mat-raised-button' directive. Set its style by adding the class 'mat-primary' or 'mat-secondary'. Or disable it with the 'disabled' attribute.

When to use

Primary buttons should be used to perform the primary action in a given context, e.g. submitting a form or advancing to the next step in a process.

Secondary buttons should be used to perform the secondary action in any given context, for example cancelling an action. They should generally be positioned to the left of the primary action with a 1rem margin between the two buttons.

Buttons should be disabled when the user cannot perform the action for which it is used. For example, if a form hasn't been filled in, the button to submit it should be disabled.
