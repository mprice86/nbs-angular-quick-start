Colour palette

At NBS, white and purple are core to our identity and appear on all communications. White space has an important role in our branding. It helps reduce clutter and creates clarity in our communications. We use white space to give our key messages the impact they deserve.

Angular theming

The nbs-material.theme file contained in this project sets the primary, accent and warning colours for all of the Angular Material components that examples have been provided for. These colours should not be overriden or changed except in special circumstances.

Core colours

NBS brand primary - should be used predominantly for H1 and background colours (e.g. on tooltips, toolbar).

NBS cta primary - should be used for calls to action such as progressive action buttons and links.

NBS text primary - should be used for body copy and all headings other than H1.

NBS utility grey - should be used for borders, disabled buttons and similar element highlights.

Additional colours

NBS add highlight - should be used as a background colour to highlight an element or text.

NBS add success - should be used to indicate an action having been successfully completed.

NBS add warning - should be used to indicate a non-critical issue to the user.

NBS add failure - should be used to indicate an error or warning to the user.

Colour combinations

For guidance on which colour combinations are appropriate, please see this Adobe XD document.

Use in code

All of the colours exist as .css classes (as shown in the code section) and are also available as .scss variables (also shown in the code section). When using these colours you should always use either the variables or the pre-written classes to avoid duplicating code.
